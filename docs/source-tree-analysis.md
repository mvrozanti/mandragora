# mandragora-nixos — Source Tree Analysis

**Date:** 2026-04-25

## Overview

The repository is a single-host NixOS flake. Every directory has one purpose,
and every file's role is implied by where it lives — that is the consequence
of the "language purity" rule, which forbids embedding non-Nix code inside
`.nix` files. Concretely: anything you might think of as "config" (Hyprland's
`.conf`, mpd's `.conf`, neovim's Lua, waybar's CSS) lives at the repo root in
its XDG-mirrored location, and the corresponding `.nix` file pulls it in via
`builtins.readFile`. This means the directory layout doubles as a map of the
running system.

## Critical Directories

### `flake.nix`

Top-level flake. Pulls `nixpkgs` (`nixos-unstable` channel), `home-manager`,
`sops-nix`, and `impermanence`. Exposes a single
`nixosConfigurations.mandragora-desktop` output. To add a new system input
(say a new flake source), this is the only place it's wired.

**Entry Point:** `nixosConfigurations.mandragora-desktop`

### `hosts/mandragora-desktop/`

The composition root. `default.nix` imports every module under `modules/` and
sets host-wide settings: `nixpkgs.config.allowUnfree`, `time.timeZone`,
`i18n.defaultLocale`, nix garbage collection schedule, systemd-boot
generation limit, and `system.stateVersion`. Adding a new module to the host
means adding a new line here.

`hardware-configuration.nix` is generated by `nixos-generate-config` at
install time and committed; do not regenerate it casually — it captures the
specific UUIDs and kernel modules the host needs.

**Entry Points:** `hosts/mandragora-desktop/default.nix`

### `modules/core/`

OS-level concerns. **One file per topic.** When you need to change boot
behavior, you go to `boot.nix` — never to a generic "system.nix" — because
there is none. The split is:

- **`globals.nix`** — `environment.systemPackages`, `nix-ld` for dynamic
  binaries, fonts, timezone-related but not timezone itself.
- **`boot.nix`** — `boot.loader.systemd-boot`, kernel package selection
  (`linuxPackages_zen`), kernel modules, kernel parameters.
- **`storage.nix`** — `fileSystems`, `swapDevices`, mount options.
- **`impermanence.nix`** — _the_ persistent paths whitelist. Anything not in
  here is wiped on boot.
- **`persistence-vms.nix`** — VM disk paths that must persist.
- **`secrets.nix`** — sops-nix module enable, age key path
  (`/persistent/secrets/keys.txt`), every secret declaration.
- **`security.nix`** — SSH daemon, firewall, polkit rules, sudo.
- **`graphics.nix`** — NVIDIA proprietary driver pin, kernel module config,
  Wayland environment variables.
- **`monitoring.nix`** — Prometheus + Grafana stack.
- **`ai-local.nix`** — Local LLM runtime.
- **`vm.nix`** — libvirt / qemu.

### `modules/desktop/`

GUI / Wayland session. Each module enables one optional desktop component:
`hyprland.nix` (the compositor), `kdeconnect.nix`, `keyledsd.nix`,
`openrgb.nix`, `seafile.nix`, `steam.nix`, `minecraft.nix`. Disabling any of
these is a one-line removal from `hosts/mandragora-desktop/default.nix`.

### `modules/user/`

home-manager scope. The user `m` is configured here. `home.nix` is the spine
— it pulls in `programs.firefox`, `programs.kitty`, theming, etc. The
peripheral `*.nix` files (`zsh.nix`, `tmux.nix`, `waybar.nix`, `lf.nix`,
`services.nix`) each handle one program and pull external config text from
`.config/<app>/` or `snippets/` via `builtins.readFile`.

**Multi-Agent Safety Note:** `home.nix` has been clobbered by full-rewrites
in the past (2026-04-20 incident dropped `programs.firefox` Tridactyl
wiring). Only ever do targeted edits here.

### `modules/audits/`

`strays.sh` is run periodically to detect imperative state — files outside
declared persistent paths, processes that don't correspond to a Nix-managed
service, etc. Discrepancies surface as warnings.

### `.config/` and `.local/bin/` and `snippets/`

The non-Nix half of the codebase. The "language purity" rule routes any
config file longer than a few lines or any code in shell/Python/Lua/CSS into
one of these directories. The corresponding `.nix` file consumes it via
`builtins.readFile ../../.config/<app>/<file>` or
`pkgs.writeShellScript "name" (builtins.readFile ../../.local/bin/<script>.sh)`.

**Pattern:** if you find yourself reaching for `extraConfig = ''…''` or
`text = ''…''` inside a `.nix` file, stop — extract the body to `.config/` or
`snippets/` and reference it.

### `pkgs/`

Local nixpkgs supplements. `overlays.nix` wires every overlay into nixpkgs.
The subdirectories are individual derivations: `claude-code/` (Anthropic
CLI), `du-exporter/` (Prometheus exporter for disk-usage metrics), `rtk/`.
Add a new local package by creating `pkgs/<name>/default.nix` and registering
it in `overlays.nix`.

### `secrets/`

sops-encrypted YAML. **Never** edited directly with a text editor — always
via `sops /etc/nixos/mandragora/secrets/secrets.yaml`. The `.sops.yaml`
configuration governs which age recipients can decrypt. The age private key
is _not_ committed; it lives at `/persistent/secrets/keys.txt`.

### `install/`

Bootstrap scripts run from a NixOS live USB to take a blank machine to
`mandragora-desktop`. `INSTALL.md` is the runbook. The shell scripts handle
disk format, subvolume creation, mount, age-key import, and finally
`nixos-install --flake`.

### `atlas/`

High-level design documents — the human-facing "why" complement to this
machine-facing "what." `PRD.md` is the requirements spec; `architecture.md`
is the design overview; `non-negotiables.md` lists hard constraints (with
`AGENTS.md` at repo root being the canonical version).

### `docs/`

This directory. AI-facing generated documentation. `project-context.md` is
the LLM-optimized rules distillate that agents load before acting.
`superpowers/` holds per-feature plans and design specs.

## Entry Points

- **System build:** `flake.nix` →
  `nixosConfigurations.mandragora-desktop` →
  `hosts/mandragora-desktop/default.nix`
- **User session:** `modules/user/home-manager.nix` → `modules/user/home.nix`
- **Boot impermanence:** systemd initrd hook defined in
  `modules/core/impermanence.nix`
- **Secrets decryption:** `modules/core/secrets.nix` consumes
  `/persistent/secrets/keys.txt` and `secrets/secrets.yaml`

## File Organization Patterns

| Pattern                                   | Purpose                                         |
| ----------------------------------------- | ----------------------------------------------- |
| `modules/<area>/<thing>.nix`              | One concern per module file; no catch-all       |
| `.config/<app>/`                          | XDG-mirrored config files for `<app>`           |
| `.local/bin/<script>.sh\|.py`             | User scripts; called by Hyprland keybinds, waybar, etc. |
| `snippets/<purpose>.<ext>`                | Code referenced from a `.nix` via `builtins.readFile` |
| `pkgs/<derivation>/default.nix`           | Local nixpkgs derivation                        |
| `atlas/<TOPIC>.md`                        | Human-facing design doc                         |
| `docs/<topic>.md`                         | AI-facing reference doc                         |

## Key File Types

### `.nix`

- **Pattern:** `*.nix`
- **Purpose:** Declarative system / home configuration
- **Examples:** `flake.nix`, `modules/core/impermanence.nix`,
  `modules/user/home.nix`

### Shell

- **Pattern:** `.local/bin/*.sh`, `snippets/*.sh`, `install/*.sh`,
  `modules/audits/strays.sh`
- **Purpose:** Anything that needs to spawn a process at runtime
- **Examples:** `.local/bin/biggest-pane.sh`, `snippets/waybar-mpd.sh`

### Python

- **Pattern:** `.local/bin/*.py`, `snippets/*.py`
- **Purpose:** Heavier scripting (HID, AI MCP server, theming helpers)
- **Examples:** `snippets/local-ai-mcp-server.py`, `.local/bin/light.py`

### Lua

- **Pattern:** `snippets/*.lua`, `snippets/keyledsd-effects/*.lua`,
  `.config/nvim/**/*.lua`
- **Purpose:** Neovim plugin glue and keyledsd effects
- **Examples:** `snippets/lightning.lua`

### CSS

- **Pattern:** `snippets/waybar-style.css`, `.config/waybar/style.css`
- **Purpose:** Waybar styling consumed via `builtins.readFile`

### Markdown

- **Pattern:** `*.md` at repo root, in `atlas/`, in `docs/`, in `install/`
- **Purpose:** Human-readable docs, design records, runbooks

## Notes for Development

- **Never `cd` into `/etc/nixos/`** to fix a runtime issue with `sudo` —
  imperative changes to that path do not survive reboot. Edit the source in
  this repo and `nixos-rebuild switch`.
- **Adding a package:** system-wide → `modules/core/globals.nix`; user-only
  → `modules/user/home.nix` `home.packages`.
- **Adding a service:** prefer a new `modules/<area>/<thing>.nix` over
  expanding an existing module past one screen.
- **Adding persistent state:** any service that writes to `/var/lib/<x>` must
  also have `<x>` added to `modules/core/impermanence.nix`. If you don't, the
  state evaporates at next boot.
- **Editing secrets:** always `sops secrets/secrets.yaml` — never open the
  file directly.
