# release-sources.txt — the "release layer" curated list
#
# One GitHub repo (owner/repo) per line. On startup the watch app
# registers a feed-only `github_release` watcher for each (push=0, so
# they appear in the watch.mvr.ac "releases" view but never ping
# Telegram). Lines are matched on (kind, target) so re-adds are no-ops.
#
# `#` starts a comment. Blank lines ignored. To stop tracking a repo,
# remove its line AND delete the watcher in the UI (the bootstrap only
# adds; it never deletes, and would re-add a still-listed repo on the
# next restart).
#
# Only works for repos that publish GitHub *Releases* (with changelog
# bodies). Tag-only projects (nixpkgs, home-manager, sops-nix, …) have
# no Releases API output — track those by adding an `rss` watcher in
# the UI pointed at https://github.com/OWNER/REPO/tags.atom instead.
